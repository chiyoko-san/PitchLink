# コントリビューションガイド

## ブランチ戦略

```
main        - 本番環境
develop     - 開発統合ブランチ
feature/*   - 新機能開発
fix/*       - バグ修正
```

## コミットメッセージ規則

```
feat: 新機能追加
fix:  バグ修正
docs: ドキュメント変更
style: コードスタイル
refactor: リファクタリング
test: テスト追加・修正
chore: ビルド・CI等
```

例：`feat: タイムラインのフィルタリング機能を追加`

## PR の出し方

1. `develop` ブランチから feature ブランチを作成
2. 変更をコミット
3. `develop` へ PR を作成
4. レビュー後マージ
